Encoder-side nonlinearity as support selection

In a sparse autoencoder, decoder directions describe what a feature writes back, while encoder-side computation determines where and when that feature fires. A residual nonlinear encoder can therefore act as a learned support-selection gate rather than only a reconstruction mechanism.

feature fires ≈ direction is useful × context says it should be active

The paper plan keeps this distinction explicit: empirical comparisons support the gate hypothesis, while broader causal claims remain gated on planned controls.
